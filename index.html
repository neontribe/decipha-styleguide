Decipha Styleguide

Headings

Only use h1 once per page, at the top of the content.

<h1><span>About us</span></h1>
<h2>A subheading</h2>
<h3>Third level heading</h3>

In this page

"In this page" is an an accessible navigation component for quickly navigating to subsections of a page. Only required on pages with numerous subsections. The URLs in the "in this page" links should point to ids on <h2> headings. For example, href="#people" going to <h2 id="people">.

<nav role="navigation">
  <h2>In this page</h2>
  <ul>
    <li><a href="#people-of-decipha">Decipha, an introduction</a></li>
    <li><a href="#decipha">The people of Decipha</a></li>
    <li><a href="#testimonial">Testimonial</a></li>
  </ul>
</nav>

Buttons

Styles for actual button elements (<button>) or "button like" elements using the .button-like class.

<!-- button -->
<button>This is a button</button>

<!-- .button-like -->
<a href="[url]" class="button-like">Button-like <a>

Text input with label

Be sure to link the label to the input's id attribute with the label's for attribute.

<label for="input-elem-id">Label for text input</label>
<input type="text" id="input-elem-id">

Product grid

Each product displays at 50% width by default but reconfigures for small screens. The images are decorative, so should have empty alt attributes as shown. The "find out more" links should link to <h2> headed sections for those products within the same page eg. href="#decipha-reading" linking to <h2 id="decipha-reading">Decipha Reading</h2>.
